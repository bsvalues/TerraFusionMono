Common Query Examples

                            
Click on any example to load it into the SQL editor.

                            
                            

                                
                                    
Basic Account Query

                                    
SELECT * FROM accounts LIMIT 100

                                
                                
                                
                                    
Find Accounts by Owner Name

                                    
SELECT * FROM accounts WHERE owner_name LIKE '%BENTON COUNTY%'

                                
                                
                                
                                    
Count Accounts by Owner Type

                                    
SELECT owner_name, COUNT(*) as count 
FROM accounts 
GROUP BY owner_name 
ORDER BY count DESC 
LIMIT 10

                                
                                
                                
                                    
Find Property Images

                                    
SELECT * FROM property_images LIMIT 100

                                
                                
                                
                                    
Parameterized Query Example

                                    
SELECT * FROM accounts WHERE owner_name = :owner_name

Query Builder Help

                
            

            

                
SQL Query Tab

                
Write SQL queries to run against the database. You can use parameterized queries with named parameters (e.g., :parameter_name) for more secure and reusable queries.

                
                
Natural Language Tab

                
Enter your query in plain English. The system will attempt to translate it to SQL which you can then execute or modify.

                
                
Examples Tab

                
Browse and select from common query examples to get started quickly.

                
                
Security Levels

                
    
                    
  • High: Blocks any potentially dangerous operations, system commands, and information disclosure.
    • 
                    
  • Medium: Allows SELECT statements but blocks data modification (INSERT, UPDATE, DELETE) and schema changes.
    • 
                    
  • Low: Only blocks the most dangerous SQL injection patterns.
    • 
                    
  • None: No security checks (use with caution).
    • 
                

                
                
Tips

                
    
                    
  • Use the Examples tab to see common query patterns.
    • 
                    
  • Use parameterized queries for better security and reusability.
    • 
                    
  • Limit your result sets to avoid performance issues.
    •
